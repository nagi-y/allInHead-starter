# Issue 作成

ユーザーが言ったアイデアを GitHub Issue に整形して登録する。

## なぜこのスキルが必要か

思いついたアイデアをそのまま放置すると揮発する。かといって自分で Issue のフォーマットを整えるのは面倒。一言投げれば、タイトル・背景・タスクリスト・完了定義・ラベルまで整えて Issue にする。記録のハードルを下げるためのスキル。

## 使い方

```
/issue-create アイデアの概要
```

---

## 手順

### 1. アイデアを受け取る

`$ARGUMENTS` がアイデアの概要。なければユーザーに「何を作る？」と聞く。

### 2. Issue を整形する

**タイトル:** 動詞 + 対象 + 目的（25字以内）

**本文フォーマット:**
```markdown
## 背景・目的

{なぜ必要か、何が嬉しくなるか}

## タスクタイプ

- [ ] `implementation` -- コード・ファイルを作成する
- [ ] `research` -- 調査・検証が主体
- [ ] `design` -- 設計・方針決定

## やること

- [ ] {具体的な作業 1}
- [ ] {具体的な作業 2}

## 完了の定義

{どうなったらこの Issue は完了か}

## 補足

{あれば: 参考リンク、制約、関連 Issue など}
```

### 3. 内容を見せてユーザーに確認

整形した Issue をユーザーに見せて「これで作っていい？」と確認する。

### 4. Issue 作成

```bash
gh issue create \
  --title "{タイトル}" \
  --body "{本文}" \
  --label "{適切なラベル}"
```

### 5. 完了報告

作成した Issue の URL をユーザーに共有する。
