# AIプロンプト設計スキル

ユーザーの要望に基づき、高品質なAIプロンプトを設計・作成する。

## 実行手順

### 1. 要件ヒアリング

- **目的**: 何を生成したいか（画像、テキスト、コード、分析等）
- **対象AI**: どのAIに入力するか（ChatGPT, Claude, Gemini, Midjourney, DALL-E等）
- **用途**: 個人利用 / 業務利用 / 販売用
- **詳細度**: ラフ指示でいいか、精密な仕様が必要か

### 2. 4層フレームワークで構造化

```
(1) 戦略レイヤー: 目的・ターゲット・訴求（なぜ作るか）
(2) 表現レイヤー: コピー・ビジュアル・配色（何を表現するか）
(3) 技術レイヤー: サイズ・フォーマット・座標・色コード（どう実装するか）
(4) 感覚レイヤー: 雰囲気・感情・トーン（どう感じさせるか）
```

### 3. テクニックの適用

| テクニック | 使いどころ | 実装方法 |
|---|---|---|
| **ロール指定** | 専門性が必要な時 | 「あなたは〇〇の専門家です」 |
| **Few-shot** | 出力形式を揃えたい時 | 例を2〜5個 |
| **Chain of Thought** | 複雑な推論が必要な時 | 「ステップごとに考えてください」 |
| **ネガティブプロンプト** | 不要な出力を防ぎたい時 | 「禁止事項:」セクション |
| **矛盾の両立指示** | 創造性を引き出したい時 | 「シンプルだが高級感がある」 |

### 4. 品質チェック

- [ ] **具体性**: 曖昧な表現が残っていないか
- [ ] **柔軟性**: AIに委ねる余地があるか
- [ ] **矛盾**: 相反する指示がないか（意図的な矛盾は除く）
- [ ] **再現性**: 同じプロンプトで毎回似た品質が出るか

### 5. 出力

1. **プロンプト本体**（コピペ可能な形式）
2. **使い方メモ**（カスタマイズポイント）
3. **改善のヒント**（出力が期待と違う時の調整方法）
