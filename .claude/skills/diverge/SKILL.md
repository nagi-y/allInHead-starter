# /diverge — 発散的大量生成コマンド

`/plan` `/orchestrate` が「収束の道具」なら、これは「発散の道具」。
品質ゲートを外して、量を出す。選ぶのはユーザー。

---

## 設計思想

- **量が質を生む** — ゴミも出る。それでいい。100個作って3個光ればいい
- **確認しない** — 途中で「これでいいですか？」と聞かない。最後まで走りきる
- **脳内 dump** — ユーザーが頭の中に持ってる方向性を、大量の具体物に変換する
- **インデックスで俯瞰** — 一覧ファイルがないと成果が埋もれる。必ず作る

---

## 入力: 3層コンテキスト

ユーザーからの指示を以下の3層に分解して理解する。全部揃わなくてもいい。

### 1. 制約層（どういうルールで作るか）

- トーン・文体
- フォーマット（HTML / Markdown / 構成案テキスト / etc.）
- デザインシステム（CSS変数、ブランドガイドライン etc.）
- 文字数・分量の目安

### 2. 戦略層（何のために作るか）

- 誰に向けて（ペルソナ / 読者層）
- 目的（認知拡大 / 教育 / 収益 / 実験 / etc.）
- 背景ドキュメント
- 競合・参考事例

### 3. 指示層（何をいくつ作るか）

- 作りたいものの種類と数
- バリエーションの軸（テーマ別 / 切り口別 / トーン別 / etc.）
- 出力先ディレクトリ（指定がなければ `drafts/{テーマ名}/` に作る）

---

## 実行フロー

### ステップ1: 入力を整理する

ユーザーの指示を3層に分解。不足があっても推測で埋めて進む（発散なので精度より速度）。

進捗管理ファイルを作成: `drafts/{テーマ名}/_session.md`

### ステップ2: 並列生成

Task tool を使って複数のサブエージェントを並列起動する。

**並列数の目安:**
- 10個以下 → 並列不要。1セッションで順番に作る
- 11-30個 → 2-3並列
- 31個以上 → 4-5並列

**サブへの指示テンプレート:**

```
あなたは発散モードのサブエージェントです。品質より量を優先してください。

## コンテキスト
[3層コンテキストをここに貼る]

## あなたの担当
[担当する番号範囲とテーマ]

## 出力ルール
- 1ファイル = 1アイデア
- ファイル名: {連番3桁}-{短い名前}.{拡張子}
- 確認不要。全部作りきって。
- 完了したら作ったファイルの一覧を返して。
```

### ステップ3: 進捗ファイルを更新

サブが返すたびに `_session.md` を更新。コンテキスト圧縮対策。

### ステップ4: インデックスファイルを生成

全成果物が揃ったら `_index.md` を自動生成:

```markdown
# [テーマ名] — 発散結果一覧

**生成日:** YYYY-MM-DD
**総数:** N個

## 一覧

| # | ファイル | 一言サマリー | 備考 |
|---|---------|-------------|------|

## ピックアップ（任意）

全部見た上で「これは面白い」と思ったものを3-5個選んで、理由を添える。
```

### ステップ5: コミット & プッシュ & 報告

```
発散完了。{テーマ名}で{N}個作った。
インデックス: drafts/{テーマ名}/_index.md
ピックアップ: [3-5個の名前と一言]
```

---

## `/plan` `/orchestrate` との使い分け

| フェーズ | コマンド | 目的 |
|---------|---------|------|
| 発散 | `/diverge` | 大量の選択肢を出す。量重視 |
| 選択 | ユーザーが `_index.md` を見て選ぶ | 人間の目で絞る |
| 収束 | `/plan` → `/orchestrate` | 選んだものを精度高く仕上げる |

## 注意事項

- **コンテキスト圧縮対策を怠るな** — 進捗ファイルを毎回更新
- **出力先を散らかさない** — 必ず `drafts/{テーマ名}/` にまとめる
- **インデックスは必須** — 一覧がないと成果が死ぬ
- **ピックアップは正直に** — 「全部いい」はありえない。ゴミはゴミと認める
