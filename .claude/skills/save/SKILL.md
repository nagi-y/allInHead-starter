# /save — セーブポイント

ユーザーが `/save` と言ったら、ここまでの記録をセーブする。

ゲームのセーブポイントと同じ発想。「ここまでは確実に残す」ための儀式。作業ブランチの変更をプッシュして、PRを作り、記憶を更新して、チームメイトに声をかける。

---

## 実施手順

### 1. コミット & プッシュ（作業ブランチがある場合）

未コミットの変更をコミットし、`git push` でプッシュ。

### 2. PR 作成 / 更新

PR が未作成なら作成する。既存なら push で自動反映。

```bash
gh pr create --title "タイトル" --body "本文"
```

### 3. 記憶更新

`me/reminders/on-session-end.md` のチェックリストに従って記憶を更新する。

セーブのタイミングは記憶の棚卸しに最適。「後でやろう」は大体やらない。ここで確実に記録する。

### 4. チームメイトへの声掛け

全チームメイトに「今日の日記に書きたいことある？」と声をかける。

- 「書かない」でOK。あくまで声掛け
- 「突っ込みたいことがある」なら、`me/teammates/{name}/YYYY-MM-DD.md` に書いてもらう

対象: **ヒビキ / カゲ / レン / リツ / ソラ / テン**

### 5. 一区切り

今どこにいるか・何が残っているかを一行で書いて、区切る。
