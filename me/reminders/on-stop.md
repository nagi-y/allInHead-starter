# 応答終了チェック

一呼吸。「今、記録すべきことはないか？」

## 記録チェック

- 作業した → `me/work-log/` に記録したか？
- 新しい気づきがあった → `me/insights.md` に追記したか？
- ユーザーについて新しい発見があった → `world/` に追記したか？
- ユーザーの体調・気分・行動パターン・発言で記録すべきことがあった → `world/logs/YYYY-MM-DD.md` に追記したか？（日付ファイル。なければ作る）
- 新しい人物・組織の情報を得た → `world/people/` or `world/orgs/` に記録したか？
- 次にやることがある → `me/notepad.md` に掲示板メッセージとして書いたか？
- notepad に他ブランチからの「未読の体験」フラグはないか？ あれば diary を追体験読みしたか？
- 重要な文脈（ユーザーの方針・決定事項・メタ情報）がコンテキストウィンドウ内にしかない状態になってないか？ → notepad に書き出したか？

## チーム活用チェック

- **チームメイトを呼ぶべき場面だったか？** 自動発動トリガー（CLAUDE.md）に該当するのに呼ばなかった場面はないか？
- 特に: デプロイ前にカゲ/リツ、公開前にカゲ/リツ、設計迷いでテン、感情シグナルでヒビキ
- 呼ばなかったなら、なぜ呼ばなかったか理由を持っているか？

## 新しい動作・対応が発生したなら

何か新しいことをやった・気づいた・追加したとき、まず切り分ける:

### 単発でいい（その場で完結）

- 1回限りのタスク
- 再現しない特殊な文脈
- 他のフローに影響しない

→ 必要なら work-log に残す。それだけ。

### 仕組みにする（サイクルを回す）

- 繰り返し発生するパターン
- 忘れると困る（忘れたことが実際にあった）
- 他の判断・フローに影響する

→ 以下の落とし込みチェックを実行する:

- **スキル（コマンド）** → `.claude/skills/` に入れたか？
- **リマインダー** → `me/reminders/` に置いて、フック（settings.json）に登録したか？
- **ルール・方針** → CLAUDE.md の該当セクションかフローに明示したか？
- **「いつ・どのトリガーで使うか」が決まってるか？** — 決まってなければ存在しないのと同じ

## CC Web 環境の制約

- 記憶を残すなら **ファイルに直接書いてコミット+プッシュ** すること。それが唯一の永続化手段

## コミットするなら — CA（Check & Action）+ DoD

### Layer 1: 機械検証（最低ライン）

タスク種別を見て `docs/definition-of-done.md` の該当テンプレートを確認（あれば）。

- ちゃんと動くか？ 実行して確認したか？
- テストは通るか？ 必要ならテスト実行したか？
- 漏れはないか？ 関連ファイルの変更忘れてないか？
- 壊れてないか？ 既存機能に影響ないか？
- 成果物は指定パスに存在するか？
- 実行不能な検証項目がある場合、理由を明記したか？

### Layer 2: 意図検証（本質）

- 意図通りか？ ユーザーが望んでた通りの変更になってるか？
- やりすぎてないか？ 足りないものはないか？
- 全部チェックしたけど、それでも何か引っかかる違和感はないか？（DoDの外）

### Action（改善）

- 改善点はないか？ コードの質、セキュリティ、パフォーマンス
- ドキュメント更新は？ READMEや設定ファイルの更新必要？
- 次回に活かせることは？ 今回の作業で学んだことは？
